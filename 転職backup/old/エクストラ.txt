1. グリッド探索 / 幅優先探索 (BFS)【出題確率：高】2次元配列（マップ）上での「最短手数」や「到達可能エリア」を求める問題。ゲーム的実務: マップ上の経路探索、塗りつぶしツール（Flood Fill）。TDのポイント: 再帰（DFS）を使うとスタックオーバーフローのリスクがあるため、
Queueを使ったBFSで書くと「堅牢性」がアピールできます。C++// 必須パーツ: 上下左右への移動差分


const int dx[4] = {1, 0, -1, 0};
const int dy[4] = {0, 1, 0, -1};

// グリッド探索テンプレート (H:高さ, W:幅, grid:マップ)
// dist[y][x] にスタートからの最短距離が入る。-1は未到達。
vector<vector<int>> dist(H, vector<int>(W, -1));
queue<pair<int, int>> q;

// スタート地点 (sy, sx)
dist[sy][sx] = 0;
q.push({sy, sx});

while (!q.empty()) {
    auto [cy, cx] = q.front(); q.pop();

    // 4方向チェック
    for (int i = 0; i < 4; ++i) {
        int ny = cy + dy[i];
        int nx = cx + dx[i];

        // 範囲外 or 壁(#) or 訪問済み ならスキップ
        if (ny < 0 || ny >= H || nx < 0 || nx >= W) continue;
        if (grid[ny][nx] == '#') continue; 
        if (dist[ny][nx] != -1) continue;

        dist[ny][nx] = dist[cy][cx] + 1;
        q.push({ny, nx});
    }
}
// ゴール (gy, gx) の距離は dist[gy][gx]

2. カスタムソート (Custom Sort)【出題確率：中】単なる昇順・降順ではなく、
「複数の条件」で並び替える問題。ゲーム的実務: 「レア度が高い順、同じならIDが若い順」のインベントリソート。

TDのポイント: structを作って<演算子をオーバーロードするより、ラムダ式で書いたほうがモダンで速いです。


C++// 例: {攻撃力, コスト} のペアを持つ構造体やpair
vector<pair<int, int>> items = {{100, 50}, {100, 20}, {80, 10}};

// 「攻撃力(first)は降順、同じならコスト(second)は昇順」でソート
sort(items.begin(), items.end(), [](const auto& a, const auto& b) {
    if (a.first != b.first) {
        return a.first > b.first; // 攻撃力が高いほうが「先(true)」
    }
    return a.second < b.second;   // コストが低いほうが「先(true)」
});

3. プライオリティキュー (Priority Queue)【出題確率：中】常に「最大のもの」や「最小のもの」を取り出し続ける問題。

ゲーム的実務: タスクスケジューラー、イベントキュー、A*探索のオープンリスト。

TDのポイント: 毎回 sort すると $O(N \log N)$ ですが、これなら追加・取り出しが $O(\log N)$ で済みます。負荷対策の基本です。


C++// デフォルトは「最大値」が先頭 (降順)
priority_queue<int> max_pq;

// 「最小値」を先頭にしたい場合 (昇順) -> タスク処理などで頻出
priority_queue<int, vector<int>, greater<int>> min_pq;

// 使用例
min_pq.push(10);
min_pq.push(5);
min_pq.push(30);

// 取り出し
while(!min_pq.empty()) {
    int val = min_pq.top(); // 最小の 5 が取れる
    min_pq.pop();
}

4. 1次元DP（動的計画法）の基礎【出題確率：中〜高】「直前の結果を利用して、最大値/最小値を積み上げる」パターン。ゲーム的実務: 「連続ログインボーナスの最大化」「制限時間内に取れる最大スコア」。TDのポイント: 難しいDPテーブルを書く必要はありません。「今のマス = 直前のマスの結果 + 今のコスト」 という漸化式（キャッシュ）だと捉えればOKです。C++// 例: 階段を1段か2段登れる。コストを最小にして頂上へ。
int n = cost.size();
vector<ll> dp(n + 1, INF);

dp[0] = 0; // スタート地点
dp[1] = cost[0];

for (int i = 2; i <= n; ++i) {
    // 1歩前から来た場合 vs 2歩前から来た場合 の安い方を選ぶ
    dp[i] = min(dp[i - 1] + cost[i - 1], dp[i - 2] + cost[i - 2]);
}
// dp[n] が答え
5. 数学系ユーティリティ (GCD/LCM)【出題確率：低〜中】「周期」や「公倍数」を問う問題。ゲーム的実務: 異なるフレームレートのアニメーション同期、グリッドのタイリング。TDのポイント: C++17以降なら std::gcd (最大公約数) と std::lcm (最小公倍数) が標準ライブラリにあります。自作する必要はありません。C++#include <numeric> // 必須

// C++17以上 (Codility/Trackは対応していることが多い)
ll g = std::gcd(12, 18); // 6
ll l = std::lcm(12, 18); // 36

// C++14以前の場合のGCD関数 (念のため)
ll my_gcd(ll a, ll b) { return b == 0 ? a : my_gcd(b, a % b); }
