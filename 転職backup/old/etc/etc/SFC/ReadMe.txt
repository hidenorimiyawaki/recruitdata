
================================================
Unoficial Snes9x v1.39 Windows port beta9.9.7pp
================================================

● これは？
Snes9x.comから落してきたソースコードを使って勝手に作った
UnofficialなSnes9x Windows portです。

本家Snes9XW.exeの3D/FX、OpenGLモードを除いた表示モードをサポートしています。
スーファミでは無意味かもしれない…連射機能を搭載しています
キャラバンシューティングコレクションくらいしか使い途は無いと思います…
コントローラー設定のTurboにチェックで使えます。
beta6と違いBS-X BIOS読み込みは無いです、すみません…。

動作環境、Windows98とかの32bit Windows OS
VC++でコンパイルした以前のは”DirectX3以上”が必要と言い切れましたが
現在のはgcc Mingw32でコンパイル（VC++コンパイルより動作が軽いみたい）
でヘッダライブラリはそれ用のDirectX6相当と
書かれていたのをどこかから落してきて入れて使っているので
必要なDirectXバージョンがいくつ以上必要なのかはわかりません
自分のWindows98デフォルト環境”DirectX5”でも
普通に動いています（DirectX6相当なのに…？？）
DirectX5よりももっと低いバージョンのDirectX環境でももしかしたら
動くのかもしれません。

（あとこのテキストの変わったところは履歴のみです）

● 使い方
まず適当なフォルダを作ってzipの中身の実行ファイル(Snes9xW.exe）をコピーしてください
実行ファイル、または作ったショートカットをダブルクリックで起動します。

まずコントローラー設定をしてください
メニューのオプション→環境設定
プロパティーシートの”コントローラー設定”を開いてください
ここで設定したいボタンのボックスをクリックしてフォーカスを合わせて
割り当てたいボタンを押してコントローラー設定をしてください。

＊自分はマイクロソフトサイドワインダーを使って一人プレイしか試していないので
＊複数コントローラーを設定、同時プレイがちゃんとできるのかどうかは未確認です。

あとはファイルメニューから遊びたいゲームのイメージをロードしてください。
ウインドウにロムファイルドラッグ＆ドロップでもＯＫです。

メニューのオプション→２倍表示 にチェックで２倍表示できます。
同じくメニューのオプション→ウインドウサイズ調整で２倍表示切り換えと同時に
ウインドウクライアント領域サイズをゲーム画面を表示する大きさに
合ったサイズに調節できます。

Shift+F1～F9でリアルタイムセーブ、
F1～F9でリアルタイムセーブロード(スロット9個)です。

（これだけわかれば知識は十分です、遊べます…）


● 使い方２

環境によって音が綺麗に鳴らない場合は
環境設定ダイアログのサウンド設定部のMixInterval、BufferLengthを
適当に変えてみてください。

"SampleDecodingMethod"を変えるとノイズ関係が変わります
Snes9x
Anti Resonance
Anti Resonance+AsmFix （デフォルト）
ZSNES
ZSNES++
UOsnes9x
と６種類あります。
Anti Resonance+AsmFixはAnti Resonance SampleDecodingMethodにちょっと手を加えた物です
聖剣伝説３、スパロボEXなどでAnti Resonanceよりちょっといいです。
UOsnes9xはちょっとあんまりよくないです…一部ノイズが鳴らせるくらいです。
場合によって（？）いろいろ変えるといいかもしれません…。

ロム読み込みダイアログで読み込むロムのタイプを強制指定できます
（本家Snes9XW.exeでもできます…）
普通に読み込んで動かないロムの場合は強制指定で動く場合があります
例えば｢ウィザードリィ外伝４｣はインターリーブ強制で
動いたりします、他にもこういうのがあると思います。
それと…ＢＳ（サテラビュー）のロムは対応していないため
SNESヘッダの認識の失敗でFPS50のPALタイミングになってしまったりするので
NTSC強制指定をするとちょっといいです。

ファイルメニューの”リセット”からリセットできます
その下の”ソフトリセット”というのはSNESワークラムをクリアしないリセットです
リセット前の状態を必要とするゲーム（かまいたち等？）で使うといいかもしれません。
この機能でゲームが期待通りの動作を本当にしてくれるのかどうかは自分では未確認です。
この機能は簡単なので付けてみました。
リセット時にワークラムをクリアしないというのがスーパーファミコンの正しい動作
なのかもしれません、Snes9xの普通のリセットではクリアしてしまいます。

customized GFX enzineはまだ（まだか…）いじり中で機能しません。

環境設定ダイアログのグラフィック設定の
”スクリーンへ直描き”、”DirectDrawオフスクリーンをVRAMへ確保”
等の設定を変えるとディスプレイモードによって動作が軽くなったりして快適に
なることがあるかもしれません、ちょこちょこ変えてみるのもいいかもしれません。

”ダブルバッファリング”、”DirectDrawバックバッファをVRAMへ確保”
等の設定はフルスクリーンモードに影響する設定です、自分で試してみたところ
変えても特に何が変わったとかよくわからない…というような感じの設定かもしれないです。

環境設定ダイアログからスレッドの優先順位、CPU占有率的なものを変えることが出来ます
詳しい意味は自分はよくわかっていないので…詳しく知りたい方は
Windowsプログラミング的なにかをお調べください、WindowsAPIを使ってそういうように
アプリケーション、プログラムの設定をする設定です…。
これらをREALTIME_PRIORITY_CLASS、THREAD_PRIORITY_TIME_CRITICAL
など高く設定するとゲーム動作中フレームレートがかなり上がります…が、
残念ながら自分の環境では60フレームいかない場合は音飛びするようになってしまいます…。
これらの設定はSPCプレイヤー時なんかには結構有効かもしれません。

● 快適に実行するために
ここにウインドウモードでの描画法指定の設定
（環境設定ダイアログのグラフィック設定でする）のヒントを書きます。

まず、自分の環境はCPUセレロン300A、ビデオカードATI 3D RAGE PROです
この環境でUOsnes9XW作って動かして遊んでます。

ウインドウモードで最もフレームレートが高く速く快適に実行できる設定は
一番よく使うと思われる２倍表示モードで…
”デスクトップ色数16bit”（この設定はウインドウズでやる設定です）
”DoubleRenderingModes HardwareX2””スクリーンへ直描き オフ”
”DirectDrawオフスクリーンをVRAMへ確保 オン”
でした。
32bit色数デスクトップでも同様の設定で一番フレームレートが高く速かったです。

24bit色数デスクトップの場合は（これも２倍表示モードとして…）
”DoubleRenderingModes SoftwareX2””スクリーンへ直描き オン”
が最も快適でした（パソコンスペックが低いのでそんな快適でもないけど…）

24bit色数デスクトップで
”DirectDrawオフスクリーンをVRAMへ確保 オン”、”DoubleRenderingModes HardwareX2”
やると転送バイト数、配置が関係しているのか…
ハードウェアでVRAMからVRAMへ（？？）転送と同時に拡大処理
（これらの設定ではそれをやると思われる…）が
なにか引っ掛かったように遅くなりました…。

参考になりますでしょうか。

● 注意点
フルスクリーンへ切り換え等ディスプレイモード切り換えに失敗した場合は
エミュレーターが終了します。

考えられますので、iniファイルの
FullscreenDisplayModeIndex（フルスクリーンで使うディスプレイモードの設定）
を変えてみてください。
